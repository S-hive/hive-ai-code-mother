# 应用生成类型下拉框设计

## 目标

将应用管理页的生成类型搜索条件由文本输入框改为下拉框，并在表格中显示对应的中文名称。生成类型集中在独立常量文件中维护。

## 常量设计

新建 `src/constants/codeGenType.ts`，导出生成类型选项数组：

- `html`：原生 HTML 模式
- `multi_file`：原生多文件模式

选项数组作为下拉框和表格显示的唯一数据源，避免重复维护。

## 页面交互

- 搜索栏使用可清空的下拉框。
- 选择类型后，仍将后端枚举值 `html` 或 `multi_file` 作为查询参数提交。
- 清空选择后不按生成类型筛选。
- 表格“生成类型”列显示对应中文名称。
- 后端返回未知类型时，表格回退显示原始值；无值时显示 `-`。

## 验证

- 类型检查和生产构建通过。
- 下拉框包含两个后端支持的生成类型。
- 搜索请求使用正确的枚举值。
- 表格正确显示中文名称及未知值回退结果。
