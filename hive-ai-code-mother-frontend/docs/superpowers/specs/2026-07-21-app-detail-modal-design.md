# 对话页应用详情弹窗设计

日期：2026-07-21

## 目标

在应用对话详情页的部署按钮左侧增加“应用详情”按钮，让用户无需离开当前页面即可查看应用创建者和创建时间，并让应用创建者或管理员执行修改、删除操作。

## 交互与布局

- 顶栏右侧将“应用详情”和“部署”按钮并排展示，“应用详情”位于左侧。
- 点击“应用详情”后打开 Ant Design Vue 模态框。
- 弹窗基础信息区从上到下展示：
  - 创建者头像和昵称；头像或昵称缺失时使用现有组件的默认展示。
  - 创建时间。
- 当前登录用户是应用创建者或管理员时，在信息区下方展示操作栏；其他用户只看到基础信息。
- 操作栏包含“修改”和“删除”按钮。

## 权限与操作

- 创建者身份通过当前登录用户 ID 与应用 `userId` 比较。
- 管理员身份通过当前登录用户 `userRole === 'admin'` 判断。
- 点击“修改”关闭弹窗并跳转到 `/app/edit/:appId`，由现有编辑页按身份处理字段和接口。
- 点击“删除”弹出二次确认：
  - 管理员调用 `/app/admin/delete`。
  - 创建者调用 `/app/delete`。
- 删除成功后关闭弹窗、提示成功并返回首页。
- 删除失败时保留当前页面和详情弹窗，并展示错误提示。

## 数据流

- 复用对话页已有的 `getAppVoById` 返回数据，不为打开弹窗增加请求。
- 创建者信息取自 `app.user`，创建时间取自 `app.createTime`。
- 登录用户信息取自现有 Pinia 登录用户 Store。

## 实现范围

- 功能直接集成在 `AppChatPage.vue`，避免为单一页面的小型弹窗增加额外组件。
- 使用现有删除接口、路由和 Ant Design Vue 组件，不新增依赖和后端接口。

## 验收

- 部署按钮左侧显示“应用详情”按钮。
- 点击后正确展示创建者头像、昵称及创建时间。
- 非创建者且非管理员看不到修改、删除按钮。
- 创建者和管理员可以进入编辑页。
- 删除必须二次确认，并按用户身份调用正确接口。
- 删除成功返回首页，失败时页面不跳转并显示错误信息。
