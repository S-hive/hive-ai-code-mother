# 对话页代码/预览切换设计

日期：2026-07-21

## 目标

将应用生成对话页升级为“左侧对话 + 右侧代码/预览切换”布局，不增加文件树和源码编辑能力。

## 布局

- 顶栏保留应用名称和部署按钮，中间增加“代码 / 预览”切换。
- 左侧约 34%：消息列表与提示词输入框。
- 右侧约 66%：根据切换状态显示只读代码查看器或 iframe 预览。
- 移动端继续使用上下布局。

## 代码查看

- `html` 类型仅显示 `index.html`。
- `multi_file` 类型显示 `index.html`、`style.css`、`script.js` 标签。
- 文件内容通过现有静态资源地址读取：
  `http://localhost:8123/api/static/{codeGenType}_{appId}/{fileName}`。
- 查看器只读，提供行号、复制按钮、横向滚动，不引入代码编辑器依赖。
- 切换文件标签时展示对应真实文件内容。

## 数据流

1. 页面加载应用详情并生成预览 URL。
2. 根据 `codeGenType` 初始化文件标签。
3. 首次进入代码模式时读取当前文件。
4. SSE 完成后刷新应用详情、iframe 和当前源码文件，避免缓存旧内容。
5. 默认保持预览模式，满足生成结束后自动展示网站效果的原需求。

## 错误处理

- 文件请求 404 或网络失败时，在代码区显示错误状态。
- 文件加载失败不影响 iframe 预览和继续对话。
- 复制成功或失败均使用 Ant Design 消息提示。

## 非目标

- 不实现文件树。
- 不允许编辑或保存源码。
- 不新增后端文件管理接口。
- 不引入 Monaco、CodeMirror 等编辑器依赖。

## 验收

- 可在代码和预览间切换。
- 多文件应用可切换三个真实文件，单文件应用只展示 HTML。
- 代码有行号，可复制，长行可横向滚动。
- 新一轮生成完成后，预览和代码内容同步刷新。
